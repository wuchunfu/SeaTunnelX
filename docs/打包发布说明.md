# SeaTunnelX 打包发布说明（MVP）

## 1. 打包目标

当前发布流程围绕以下目标：

- 安装包支持 `amd64/arm64` 双架构；
- Node 运行时以 **Node 22** 为主，发布 `official` 与 `glibc217` 两个变体；
- 每个变体同时产出：
  - `with-observability`（内置三件套）
  - `without-observability`（不内置三件套）
- 安装包内置：
  - `install.sh`（一键安装）
  - `bin/start.sh` / `bin/stop.sh` / `bin/status.sh`

---

## 2. 本地打包命令

主脚本：`scripts/package-release.sh`

```bash
# Node 22 主矩阵（官方 Node）
scripts/package-release.sh --arch all --bundle-observability both --node-major 22 --node-variant official --build-frontend

# Node 22 glibc2.17 兼容变体（CentOS7 等）
scripts/package-release.sh --arch all --bundle-observability both --node-major 22 --node-variant glibc217 --build-frontend
```

产物默认输出到：`dist/releases/`

命名示例：

```text
seatunnelx-v1.2.0-linux-amd64-node22-official-with-observability.tar.gz
seatunnelx-v1.2.0-linux-arm64-node22-glibc217-without-observability.tar.gz
```

---

## 3. 安装包使用方式（用户侧）

```bash
tar -xzf seatunnelx-xxx.tar.gz
cd seatunnelx-xxx

# 一键安装（默认安装到 /opt/seatunnelx，并自动启动）
./install.sh

# 自定义安装目录，不自动启动
./install.sh --install-dir /data/seatunnelx --no-start
```

安装后常用命令：

```bash
/opt/seatunnelx/bin/start.sh
/opt/seatunnelx/bin/stop.sh
/opt/seatunnelx/bin/status.sh
```

## 4. GitHub Actions 流程

### 4.1 main 分支 CI

工作流：`.github/workflows/ci-main.yml`

触发：
- `push` 到 `main`
- `pull_request` 到 `main`

内容：
- 后端 + Agent 测试（`go test`）
- 前端测试 + 构建（Node 22）

### 4.2 tag 发布

工作流：`.github/workflows/release-on-tag.yml`

触发：
- `push tag`，模式：`v*`

内容：
1. 按 Node22 变体矩阵打包并上传 artifacts；
2. 自动创建/更新 GitHub Release 并挂载安装包；
3. 自动发布 GHCR 镜像：
   - `ghcr.io/<owner>/seatunnelx-backend:<tag>`
   - `ghcr.io/<owner>/seatunnelx-frontend:<tag>`
   - `ghcr.io/<owner>/seatunnelx-all-in-one:<tag>`

## 5. All-in-one 镜像说明

`seatunnelx-all-in-one` 镜像包含：

- 后端 `seatunnelx api`
- 前端 Next.js standalone
- Node 22 runtime

启动后默认监听：
- 前端：`80`
- 后端：`8000`
- gRPC：`9000`

入口脚本：`.github/docker/all-in-one-entrypoint.sh`
