# SeaTunnelX 可观测性三件套一键接入说明（MVP）

## 1. 设计原则（按当前版本）

- 本地 `deps` 脚本只负责本地三件套启停与状态检查；
- 不再提供“本地脚本自动切远程模式/自动改写远程配置”；
- 远程联动统一走 `config.yaml` + 外部三件套手工配置；
- 默认地址即本地回环：`127.0.0.1:9090/9093/3000`。

---

## 2. 场景 A：使用本地 deps 三件套（一键）

### 2.1 默认方式（推荐）

直接启动：

```bash
./deps/start-observability.sh
./deps/status-observability.sh
```

说明：
- `start-observability.sh` 默认会执行配置初始化（`OBSERVABILITY_AUTO_INIT=true`）；
- 适合“本机部署三件套 + 默认配置”场景。

### 2.2 预置配置文件方式（你提出的方式）

如果你希望使用自己准备好的配置文件，只需把文件放到 `deps/runtime`，并关闭自动初始化：

```bash
# 你自己的配置文件（示例）
deps/runtime/prometheus/prometheus.yml
deps/runtime/grafana/grafana.ini
# 可选：deps/runtime/alertmanager/alertmanager.yml

# 使用预置配置启动（不覆盖）
OBSERVABILITY_AUTO_INIT=false ./deps/start-observability.sh
./deps/status-observability.sh
```

> 这就是“本地放好 prometheus/grafana 配置，一键启动引入”的最小实现方式。

---

## 3. 场景 B：已有外部三件套（不使用本地 deps 托管）

只做两件事：

1) 配置 SeaTunnelX 的三件套地址（`config.yaml`）：

```yaml
observability:
  enabled: true
  prometheus:
    url: "http://127.0.0.1:9090"
    http_sd_path: "/api/v1/monitoring/prometheus/discovery"
  alertmanager:
    url: "http://127.0.0.1:9093"
    webhook_path: "/api/v1/monitoring/alertmanager/webhook"
  grafana:
    url: "http://127.0.0.1:3000"
```

2) 让外部三件套按接入规范手工修改配置（HTTP SD / webhook / datasource/dashboard）。

---

## 4. 一键检查清单

- SeaTunnelX 健康：`GET /api/v1/health`
- HTTP SD：`GET /api/v1/monitoring/prometheus/discovery`
- Webhook：`POST /api/v1/monitoring/alertmanager/webhook`
- 可选 smoke：

```bash
SEATUNNELX_USERNAME=admin \
SEATUNNELX_PASSWORD=admin \
./scripts/observability-remote-smoke.sh https://your-seatunnelx.example.com
```
